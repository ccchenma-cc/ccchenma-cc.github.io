Problem and opportunity

Most desktop-pet products are visually cute but thin as systems: they can appear on screen, yet rarely carry a stable personality, world logic, dialogue style, or long-term interaction promise. Lumi Galaxy explores whether a small AI-native character universe can make desktop companionship feel more memorable and extensible.

The core opportunity is not only "make a cute pet", but build a reusable character framework that can support different personalities, dialogue tones, visual states, and future memory.

The product needed a narrow MVP: prove the universe, role differences, basic interaction, and launch packaging before expanding into deeper AI companion features.

Product strategy

The first version was positioned as a creative AI companion prototype, not a complete productivity tool. The priority was to validate whether the characters, world setting, and desktop/web experience could form a coherent product direction.

MVP

Start with the universe

Define the world, character set, visual identity, and interaction promise before adding heavy AI memory or personalization.

UX

Make roles distinct

Each companion needs a different emotional rhythm, speaking style, and use scenario so the system does not feel like one generic assistant wearing five skins.

Iteration

Leave room to scale

The data-driven character structure keeps the next steps open: more dialogue, persistent memory, richer desktop states, and themed character expansion.

Character system

The character layer is designed as a product system: each alien has an identity, personality, PAD emotion baseline, speaking style, visual assets, and dialogue direction.

Spaco

Curious explorer

High curiosity and light energy; short sentences, sound effects, and a playful habit of reframing human life as alien-world logic.

Daise

Gentle caretaker

Slow, soft, and caring; built for low-pressure companionship, water reminders, and calm emotional support.

Lumi

Focused coding partner

Precise and slightly dry; positioned as a calm companion for work sessions, coding focus, and quiet productivity.

AI production workflow

AI tools were used as production accelerators, while the product decisions stayed human-led: what the product should be, how each character should differ, what to keep in the MVP, and when the output no longer matched the world direction.

Concept and positioning
Character writing and visual drafts
Dialogue data and frontend prototype
Manual direction, pruning, and launch

AI helped produce character background drafts, pixel-art directions, dialogue variants, page implementation, and iteration options.

The product judgment stayed in the selection layer: choosing the product boundary, unifying tone, rejecting inconsistent assets, and shaping the roadmap.

Product architecture

The case combines a visual layer, a character layer, and a knowledge layer. That structure shows how the project can evolve from a portfolio website into a richer AI companion product with dialogue and persistence.

Display

Web planet and desktop surface

A public web case and a desktop companion shell make the product visible, shareable, and easy to inspect.

Character

Configurable companion data

Character profiles, emotional baselines, assets, and dialogue rules are separated from the interface so the universe can expand.

Interaction

Dialogue, menus, states, and mini-games

The experience includes basic interaction hooks that can later connect to memory, personalization, and richer AI behavior.

Iteration roadmap

The roadmap separates what has already been validated from what should come next, so the case reads like an evolving product rather than a one-time visual experiment.

V0

World, role, and website

Define the universe, ship the visual web page, and make the project externally readable.

V1

Desktop pet and interaction shell

Support desktop presence, menus, state changes, installation notes, and basic playful interactions.

V2

Dialogue and emotional response

Expand intent recognition, character-specific replies, PAD emotion changes, and small interaction loops.

V3

Memory and character growth

Add user preferences, long-term memory, relationship progression, and richer companion behavior.

Outcome

Demonstrated the feasibility of using a pure AI toolchain to move from creative concept to a launched product prototype. The project shows solo 0-to-1 product shaping, AI-assisted content production, and the ability to turn a playful idea into a polished web artifact.
